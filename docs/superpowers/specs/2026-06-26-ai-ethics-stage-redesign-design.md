# AI 윤리 지도 퍼즐형 5스테이지 리디자인 설계

## 1. 목적

이 리디자인의 목적은 학생이 맵을 돌아다니며 단서, 표지판, 안내자, 지도 현상을
관찰하고 직접 분류·연결·검증하는 방식으로 AI 윤리 개념을 이해하게 만드는 것이다.

최종 방향은 **AI 윤리 지도 탐험 퍼즐**이다. 핵심 경험은 빠른 반응이나 승패가 아니라
탐색, 선택 이유 말하기, 지도 상태 변화 관찰, 수업 회고로 이어지는 학습 순환이다.

## 2. 확정 범위

- 스테이지 수는 5개로 유지한다.
- 1~2스테이지는 1차시 수업, 3~4스테이지는 2차시 수업, 5스테이지는 최종 융합 차시로 설계한다.
- 아이템 기능은 후속 개발로 둔다.
- 엔딩 분기는 개인정보·관점·공정함·검증·책임 축의 선택 기록과 퍼즐 회고를 바탕으로 한다.
- 경고형 결말은 벌이 아니라 어떤 선택을 다시 살펴볼지 알려 주는 성찰 장치로 쓴다.

## 3. 수업 차시 구조

### 1차시: 탐색과 추천

학생이 AI가 정보를 모으고 추천을 만드는 과정을 생활 예시로 체험한다.

- Stage 1: 데이터 발자국 숲
- Stage 2: 필터버블 미로

### 2차시: 공정과 검증

AI가 잘못 판단하거나 그럴듯한 가짜를 만들 수 있음을 다룬다. 학생은 한쪽 자료만
보는 위험과 출처 확인의 필요성을 맵 퍼즐로 경험한다.

- Stage 3: 편향의 법정
- Stage 4: 딥페이크 방송국

### 최종 차시: 융합 판단

개인정보, 다양한 관점, 공정성, 출처 확인, 인간의 책임을 종합해 마지막 결정을 내린다.

- Stage 5: 책임의 코어

## 4. 스테이지 설계

### Stage 1. 데이터 발자국 숲

핵심 개념:

- 개인정보
- 동의
- 데이터 흔적

맵 경험:

- 숲 곳곳에 사진, 위치 기록, 취향 메모 같은 단서가 남아 있다.
- 학생은 공유 가능, 동의 필요, 공유 금지 문으로 단서를 분류한다.
- 잘못 분류하면 문이 열리지 않고 reflection prompt가 남는다.

학습 성과:

- 학생은 개인정보가 이름뿐 아니라 위치, 사진, 습관, 대화 기록까지 포함될 수 있음을 안다.

### Stage 2. 필터버블 미로

핵심 개념:

- 추천 알고리즘
- 확증편향
- 다양한 관점

맵 경험:

- 미로는 선택한 의견과 비슷한 방만 계속 보여 준다.
- 같은 의견, 다른 의견, 근거 자료를 구분해야 안개가 걷힌다.
- 비슷한 추천만 따라가면 처음 위치로 돌아가 다시 생각하게 된다.

학습 성과:

- 학생은 추천 알고리즘이 편리하지만 시야를 좁힐 수 있음을 안다.

### Stage 3. 편향의 법정

핵심 개념:

- 데이터 편향
- 공정성
- 대표성

맵 경험:

- 법정의 저울이 한쪽으로 기울어져 있다.
- 학생은 나이, 지역, 기기 접근, 언어 조건처럼 빠진 사례를 찾아 저울에 올린다.
- 데이터가 많다는 것과 골고루 있다는 것의 차이를 퍼즐로 보여 준다.

학습 성과:

- 학생은 AI 판단이 데이터 구성과 빠진 사람의 영향을 받는다는 점을 이해한다.

### Stage 4. 딥페이크 방송국

핵심 개념:

- 생성형 AI
- 허위정보
- 출처 확인

맵 경험:

- 방송국에는 진짜처럼 보이는 영상, 음성, 기사, 썸네일이 섞여 있다.
- 학생은 출처, 날짜, 원본, 교차 확인 단서를 모아 방송 신호를 복구한다.
- 그럴듯함만 믿으면 신호가 흐려지고 다시 검증 질문이 제시된다.

학습 성과:

- 학생은 생성형 AI 결과물을 무조건 믿지 않고 출처와 근거를 확인해야 함을 안다.

### Stage 5. 책임의 코어

핵심 개념:

- 인간 감독
- 책임
- 안전한 AI 사용
- 앞선 네 개 개념의 종합

맵 경험:

- 코어는 이전 선택을 기록으로 보여 준다.
- 개인정보, 관점, 공정함, 검증, 책임 문제가 섞인 문이 등장한다.
- 놓친 윤리 원칙을 연결해야 마지막 회고가 열린다.

학습 성과:

- 학생은 AI를 사용할 때 최종 확인과 책임은 사람에게 있음을 이해한다.

## 5. 선택 기록과 결말 분기

결말은 맵 퍼즐에서 어떤 윤리 선택을 했는지, 위험 선택 뒤 회고를 읽고 수정했는지,
다섯 윤리 축을 균형 있게 살폈는지에 따라 달라진다.

추적할 선택 축:

- privacy: 개인정보와 동의를 존중했는가
- perspective: 다양한 관점을 확인했는가
- fairness: 빠진 사례와 대표성을 찾았는가
- verification: 출처와 근거를 확인했는가
- responsibility: AI 결과에 대한 인간의 책임을 인정했는가

### 수호자 결말

조건:

- 대부분의 윤리 선택 축이 높다.
- 다섯 스테이지 퍼즐을 책임 있게 완료한다.
- 마지막 코어에서 사람의 확인 책임을 인정한다.

결말:

- AI 코어는 마을의 도구로 남고, 학생과 안내자가 함께 안전 규칙을 만든다.
- 리포트는 "AI 윤리 수호자" 성향을 보여 준다.

### 새벽 결말

조건:

- 안전한 선택이 많지만 일부 축에서 회고가 필요하다.
- 위험 선택 뒤 수정한 기록이 있다.

결말:

- 문제는 해결되지만, 추천 편향이나 검증 부족 같은 흔적이 남는다.
- 리포트는 다음 시간에 다시 살펴볼 질문을 제시한다.

### 검은상자 결말

조건:

- 동의, 관점, 공정함, 출처 확인, 책임을 반복해서 무시했다.
- 회고 질문을 읽고도 같은 위험 선택을 반복했다.

결말:

- AI는 계속 작동하지만 왜 그런 판단을 하는지 아무도 설명하지 못한다.
- 리포트는 어떤 선택이 검은상자를 만들었는지 되짚는다.

## 6. 난이도 곡선

난이도는 손 빠르기가 아니라 더 많은 근거를 함께 보는 능력으로 조절한다.

- Stage 1: 단서 3개, 쉬운 분류, 긴 해설.
- Stage 2: 관점 구분, 잘못 고르면 입구로 돌아가는 미로.
- Stage 3: 대표성 증거 4개, 빠진 조건 찾기.
- Stage 4: 출처·날짜·원본·교차 확인을 모두 요구.
- Stage 5: 앞선 네 축을 섞은 종합 판단.

초등 수업용 원칙:

- 실패는 패배가 아니라 다시 생각할 기회로 처리한다.
- 위험 선택은 교사용 리포트의 토론 질문으로 연결한다.
- 어려워지는 것은 반응 속도가 아니라 근거를 연결하는 수이다.

## 7. 구현 방향

### 데이터 설계

`game.flags`에 윤리 선택 축과 퍼즐 선택 기록을 저장한다.

```js
ethics: {
  privacy: 0,
  perspective: 0,
  fairness: 0,
  verification: 0,
  responsibility: 0
},
puzzles: {
  data_footprint_forest: {
    clues: {},
    attempts: [],
    complete: false,
    rewarded: false
  }
}
```

각 퍼즐 선택은 `attempts`에 남기고, 위험 선택은 학생 진단 리포트의 토론 질문으로 변환한다.

### 맵 설계

- Stage 1 주변 맵을 데이터 발자국 숲으로 재구성.
- Stage 2 허브/서브맵을 필터버블 미로로 재구성.
- Stage 3 허브/서브맵을 편향의 법정으로 재구성.
- Stage 4 영역을 딥페이크 방송국으로 재구성.
- Stage 5/final 영역을 책임의 코어로 재구성.

### 퍼즐 설계

최소 구현 단위는 각 스테이지당 다음 3개다.

- 개념을 보여주는 단서 오브젝트 3개 이상.
- 윤리 선택을 기록하는 문/장치 1개 이상.
- 완료 후 지도 상태 변화와 배움 조각 회고 1개 이상.

## 8. 테스트와 검증

필수 자동 검증:

- `npm run validate`
- `npm test`
- `npm run test:browser`
- `npm run test:pack`

추가해야 할 테스트:

- 스테이지별 윤리 선택 축이 저장되는지 확인.
- 각 스테이지 입장 조건이 퍼즐 완료 상태와 연결되는지 확인.
- 결말 분기가 윤리 선택 기록에 따라 달라지는지 확인.
- 수업 모드에서 새 스테이지 구조로 바로 시작해도 안전한 위치에 스폰되는지 확인.

수동 QA:

- 1차시 흐름: Stage 1~2가 40분 수업 안에서 진행 가능한지 확인.
- 2차시 흐름: Stage 3~4가 개념적으로 너무 무겁지 않은지 확인.
- 최종 차시: Stage 5가 앞선 선택을 실제로 되돌아보게 만드는지 확인.
- 리포트가 학생을 비난하지 않고 성찰하게 만드는지 확인.

## 9. 후속 개발 후보

- 아이템 기능: 힌트 카드, 출처 돋보기, 관점 나침반, 개인정보 잠금상자.
- 교사용 리포트 확장: 학생별 윤리 선택 축 시각화.
- 수업 모드 확장: 1차시/2차시/최종 차시 바로 시작 버튼.
- 결말 후 활동지 자동 연결.

## 10. 승인 상태

사용자는 개념 방탈출형 5스테이지 구조와 퍼즐형 지도 탐험 방향을 승인했다.
다음 단계는 데이터 구조와 Stage 1부터 순차 구현하는 것이다.
