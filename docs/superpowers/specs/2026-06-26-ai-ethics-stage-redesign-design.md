# AI 윤리 방탈출형 5스테이지 리디자인 설계

## 1. 목적

현재 게임은 AI 윤리 퀴즈와 전투가 중심이고, 맵 자체는 윤리 개념을 체험하게 만드는 힘이 약하다. 이 리디자인의 목적은 각 스테이지를 하나의 AI 윤리 개념으로 재정의하고, 플레이어가 맵을 탐색하며 단서와 장치를 풀어 개념을 이해하게 만드는 것이다.

최종 방향은 "언더테일형 AI 윤리 RPG"다. 전투에서 이기는 것보다 몬스터의 오해를 풀고, 맵에서 내린 선택이 누적되어 결말이 달라지는 구조를 목표로 한다.

## 2. 확정 범위

- 스테이지 수는 5개로 유지한다.
- 1~2스테이지는 1차시 수업, 3~4스테이지는 2차시 수업, 5스테이지는 최종 융합 차시로 설계한다.
- 아이템 기능은 이번 리디자인의 핵심 범위에서 제외하고 후속 개발로 둔다.
- 엔딩 분기는 윤리 선택 기록 70%, 몬스터 설득 30% 비중으로 설계한다.
- "나쁜 엔딩"은 벌이 아니라 성찰형 경고 엔딩으로 만든다.

## 3. 수업 차시 구조

### 1차시: 탐색과 추천

1차시는 학생이 AI가 정보를 모으고 추천을 만드는 과정을 체험한다. 난이도는 낮고, 개념은 생활 예시 중심으로 전달한다.

- Stage 1: 데이터 발자국 숲
- Stage 2: 필터버블 미로

### 2차시: 공정과 검증

2차시는 AI가 잘못 판단하거나 그럴듯한 가짜를 만들 수 있음을 다룬다. 학생은 한쪽 자료만 보는 위험과 출처 확인의 필요성을 맵 퍼즐로 경험한다.

- Stage 3: 편향의 법정
- Stage 4: 딥페이크 방송국

### 최종 차시: 융합 판단

최종 차시는 앞선 네 개 개념을 섞어 낸다. 학생은 개인정보, 다양한 관점, 공정성, 출처 확인, 인간의 책임을 종합해 마지막 결정을 내린다.

- Stage 5: 책임의 코어

## 4. 스테이지 설계

### Stage 1. 데이터 발자국 숲

핵심 개념:

- 개인정보
- 동의
- 데이터 흔적

맵 경험:

- 숲 곳곳에 발자국, 사진, 위치 기록, 대화 기록 같은 단서가 남아 있다.
- 플레이어는 어떤 정보가 안전하게 공유 가능한지, 어떤 정보는 동의가 필요한지 분류한다.
- 잘못 분류하면 문이 열리지 않거나 NPC가 불안해한다.

방탈출 장치:

- "공유 가능", "동의 필요", "공유 금지" 세 문으로 단서를 나누는 퍼즐.
- NPC의 대사를 통해 같은 정보라도 맥락에 따라 위험도가 달라짐을 보여준다.

보스:

- 기록몬
- 오해: "데이터는 많이 모을수록 무조건 좋다."
- 난이도: 매우 쉬움. 단일 패턴, 긴 회피 시간, 명확한 정답 해설.

학습 성과:

- 학생은 개인정보가 이름뿐 아니라 위치, 사진, 습관, 대화 기록까지 포함될 수 있음을 안다.

### Stage 2. 필터버블 미로

핵심 개념:

- 추천 알고리즘
- 확증편향
- 다양한 관점

맵 경험:

- 미로는 플레이어가 선택한 의견과 비슷한 방만 계속 보여준다.
- 같은 색 방만 따라가면 출구가 사라지고 처음 위치로 돌아온다.
- 서로 다른 관점의 조각을 모아야 중앙 문이 열린다.

방탈출 장치:

- "내 생각과 같은 의견", "반대 의견", "근거가 있는 의견"을 구분하는 방 퍼즐.
- 같은 의견만 모으면 보스 방 입구가 잠긴다.

보스:

- 거울몬
- 오해: "내 화면에 많이 보이는 말이 세상의 전부다."
- 난이도: 쉬움에서 보통. 좌우 압박 패턴과 짧은 선택 제한을 도입한다.

학습 성과:

- 학생은 추천 알고리즘이 편리하지만 시야를 좁힐 수 있음을 안다.

### Stage 3. 편향의 법정

핵심 개념:

- 데이터 편향
- 공정성
- 대표성

맵 경험:

- 법정의 저울이 한쪽으로 기울어져 있다.
- 플레이어는 빠진 사례와 소외된 데이터를 찾아 저울에 올린다.
- 한쪽 데이터만 넣으면 길이 기울어지고, 특정 NPC가 계속 불리한 판정을 받는다.

방탈출 장치:

- 네 개의 증거 방에서 누락된 사례를 찾는다.
- "데이터가 많다"와 "데이터가 골고루 있다"의 차이를 퍼즐로 보여준다.

보스:

- 편향몬
- 오해: "데이터가 많으면 AI 판단은 무조건 공정하다."
- 난이도: 보통. 빈틈 벽 패턴, 함정 선택지, 반례 찾기 퀴즈를 사용한다.

학습 성과:

- 학생은 AI 판단이 데이터의 구성에 영향을 받는다는 점을 이해한다.

### Stage 4. 딥페이크 방송국

핵심 개념:

- 생성형 AI
- 허위정보
- 출처 확인

맵 경험:

- 방송국에는 진짜처럼 보이는 영상, 음성, 기사, 썸네일이 섞여 있다.
- 플레이어는 날짜, 출처, 맥락, 원본 자료 단서를 대조해 진짜 방송 신호를 복구한다.
- 그럴듯함만 보고 고르면 가짜 방으로 이동한다.

방탈출 장치:

- "출처", "날짜", "원본", "교차 확인" 네 단서를 맞춰 송출 장치를 켠다.
- NPC가 "친구가 보냈으니 진짜" 같은 흔한 착각을 말한다.

보스:

- 합성몬
- 오해: "진짜처럼 보이면 진짜다."
- 난이도: 어려움. 지그재그 패턴, 가짜 신호, 시간 제한 선택을 조합한다.

학습 성과:

- 학생은 생성형 AI 결과물을 무조건 믿지 않고 출처와 근거를 확인해야 함을 안다.

### Stage 5. 책임의 코어

핵심 개념:

- 인간 감독
- 책임
- 안전한 AI 사용
- 앞선 네 개 개념의 종합

맵 경험:

- 코어는 플레이어의 이전 선택을 기록으로 보여준다.
- 개인정보, 필터버블, 편향, 딥페이크 문제를 섞은 문이 등장한다.
- 놓친 윤리 원칙을 복구해야 최종 보스 방이 열린다.

방탈출 장치:

- "누가 확인해야 했는가", "어떤 정보를 더 봐야 했는가", "어떤 선택이 안전했는가"를 묻는 종합 퍼즐.
- 이전 스테이지에서 안전한 선택을 많이 했다면 일부 문이 자동으로 열리고, 편리함만 택했다면 추가 성찰 퍼즐이 나온다.

보스:

- 코어몬
- 오해: "AI가 한 일이니 사람은 책임이 없다."
- 난이도: 최종. 3페이즈 복합 패턴.

페이즈 구조:

1. 개인정보 페이즈: 데이터 발자국이 쏟아지는 패턴.
2. 관점/편향 페이즈: 한쪽 벽과 거울탄이 플레이어를 몰아간다.
3. 검증/책임 페이즈: 진짜 신호와 가짜 신호를 구분하며 회피한다.

학습 성과:

- 학생은 AI를 사용할 때 최종 책임과 확인은 사람에게 있음을 이해한다.

## 5. 선택 기록과 엔딩 분기

엔딩은 단순히 전투 승리 여부로 갈라지지 않는다. 맵 퍼즐에서 어떤 윤리 선택을 했는지, 몬스터를 어떻게 대했는지, 해설을 읽고 수정했는지가 함께 반영된다.

권장 비중:

- 윤리 선택 기록: 70%
- 몬스터 설득/자비 기록: 30%

추적할 선택 축:

- privacy: 개인정보와 동의를 존중했는가
- perspective: 다양한 관점을 확인했는가
- fairness: 빠진 사례와 편향을 찾았는가
- verification: 출처와 근거를 확인했는가
- responsibility: AI 결과에 대한 인간의 책임을 인정했는가
- mercy: 몬스터를 제거보다 설득으로 대했는가

### 수호자 엔딩

조건:

- 대부분의 윤리 선택 축이 높다.
- 보스와 주요 몬스터를 설득했다.
- 최종 코어에서 책임 있는 선택을 한다.

결말:

- AI 코어는 마을의 도구로 남고, 학생과 NPC가 함께 안전 규칙을 만든다.
- 리포트는 "AI 윤리 수호자" 성향을 보여준다.

### 편리함 엔딩

조건:

- 빠르고 쉬운 선택을 자주 했지만, 치명적인 무시는 적다.
- 일부 몬스터는 설득했지만 일부는 건너뛰거나 무시했다.

결말:

- 문제는 해결되지만, 추천 편향이나 검증 부족 같은 흔적이 남는다.
- 리포트는 "편리함과 안전 사이에서 더 확인할 점이 있음"을 보여준다.

### 검은상자 엔딩

조건:

- 동의, 관점, 공정성, 출처 확인, 책임을 반복해서 무시했다.
- 보스를 단순히 이기는 데 집중하고 해설/성찰 선택을 적게 했다.

결말:

- AI는 계속 작동하지만, 왜 그런 판단을 하는지 아무도 설명하지 못한다.
- 리포트는 벌을 주기보다 "어떤 선택이 검은상자를 만들었는지"를 되짚는다.

## 6. 난이도 곡선

보스 난이도는 게임오버를 강하게 만드는 방식이 아니라, 패턴 밀도와 판단 복합도를 높이는 방식으로 올린다.

- Stage 1: 단일 패턴, 느린 탄, 긴 해설.
- Stage 2: 좌우 압박, 관점 선택.
- Stage 3: 빈틈 벽, 반례 찾기.
- Stage 4: 지그재그, 가짜 신호, 시간 제한.
- Stage 5: 3페이즈 복합 패턴, 이전 선택 기록 반영.

초등 수업용 원칙:

- 하트는 기존처럼 최소 1 아래로 떨어지지 않게 유지한다.
- 실패는 패배가 아니라 다시 생각할 기회로 처리한다.
- 어려워지는 것은 손 빠르기가 아니라 "더 많은 근거를 함께 보는 능력"이어야 한다.

## 7. 구현 방향

### 데이터 설계

`game.flags`에 윤리 선택 축을 저장한다.

예상 구조:

```js
ethics: {
  privacy: 0,
  perspective: 0,
  fairness: 0,
  verification: 0,
  responsibility: 0,
  mercy: 0
}
```

각 퍼즐/대화/보스 선택은 해당 축을 증가시키거나, 무시 선택을 별도 카운트로 기록한다.

### 맵 설계

기존 허브-서브맵-보스 구조는 재사용하되, 이름과 목적을 바꾼다.

- 기존 Stage 1 주변 맵을 데이터 발자국 숲으로 재구성.
- 기존 Stage 2 허브/서브맵을 필터버블 미로로 재구성.
- 기존 Stage 3 허브/서브맵을 편향의 법정으로 재구성.
- 기존 Stage 4 영역을 딥페이크 방송국으로 재구성.
- 기존 Stage 5/final 영역을 책임의 코어로 재구성.

### 퍼즐 설계

최소 구현 단위는 각 스테이지당 다음 3개다.

- 개념을 보여주는 단서 오브젝트 3개 이상.
- 윤리 선택을 기록하는 문/장치 1개 이상.
- 보스 입장 조건 1개.

### 보스 설계

`BOSS_ATTACKS`에 stage 또는 difficulty 값을 추가해 단계별 난이도를 조정한다. 기존 패턴은 재사용하되 duration, 탄 밀도, 페이즈 수를 스테이지에 따라 다르게 둔다.

## 8. 테스트와 검증

필수 자동 검증:

- `npm run validate`
- `npm test`
- `npm run test:browser`
- `npm run test:pack`

추가해야 할 테스트:

- 스테이지별 윤리 선택 축이 저장되는지 확인.
- 각 스테이지 보스 입장 조건이 퍼즐 완료 상태와 연결되는지 확인.
- 엔딩 분기가 윤리 선택 기록과 몬스터 설득 기록에 따라 달라지는지 확인.
- 수업 모드에서 새 스테이지 구조로 바로 시작해도 안전한 위치에 스폰되는지 확인.

수동 QA:

- 1차시 흐름: Stage 1~2가 40분 수업 안에서 진행 가능한지 확인.
- 2차시 흐름: Stage 3~4가 개념적으로 너무 무겁지 않은지 확인.
- 최종 차시: Stage 5가 앞선 선택을 실제로 되돌아보게 만드는지 확인.
- 엔딩 리포트가 학생을 비난하지 않고 성찰하게 만드는지 확인.

## 9. 후속 개발 후보

이번 리디자인 이후에 추가할 수 있는 기능:

- 아이템 기능: 힌트 카드, 출처 돋보기, 관점 나침반, 개인정보 잠금상자.
- 교사용 리포트 확장: 학생별 윤리 선택 축 시각화.
- 수업 모드 확장: 1차시/2차시/최종 차시 바로 시작 버튼.
- 엔딩 후 활동지 자동 연결.

## 10. 승인 상태

사용자는 A안, 즉 개념 방탈출형 5스테이지 구조를 승인했다. 또한 추천안대로 엔딩 분기는 윤리 선택 기록 70%, 몬스터 설득 30% 방향으로 진행한다.

다음 단계는 이 설계를 구현 계획으로 쪼갠 뒤, 데이터 구조와 Stage 1부터 순차 구현하는 것이다.
