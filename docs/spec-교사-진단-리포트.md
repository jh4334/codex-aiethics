# 스펙: 교사용 학생 진단 리포트 (U3)

> spec-first 산출물. (ouroboros `init`의 인터뷰는 이 루트 환경에서 막혀서 —
> claude CLI의 `--dangerously-skip-permissions`가 root에서 거부됨 — 동일한
> 형식의 명세를 수기로 작성함. 일반 개발 머신에서는 `ouroboros init start`로 생성 가능.)

## 목표 (Goal)
교사가 한 기기를 나눠 쓰는 학생(슬롯)들의 학습 데이터를 바탕으로, **약점 주제 →
추천 차시·활동지**가 자동으로 매칭된 **인쇄 가능한 진단 리포트**를 게임 안에서
바로 만들 수 있게 한다.

## 사용자 가치 (Why)
- 대시보드·CSV는 "현황"을 보여 주지만, 교사는 "그래서 무엇을 가르칠지"가 필요하다.
- 약점 주제를 `docs/차시별-활동지.md`의 차시와 연결해 다음 수업을 바로 설계.

## 범위 (Scope)
- 입력: 슬롯별 `getStats`(주제별 정답/시도), `getMistakes`(복습 노트), 진행 단계.
- 출력: 학생 1명 또는 반 전체의 진단 리포트(화면 + 인쇄/캡처용 텍스트, CSV 보완).
- 매칭: 주제 키 → 추천 차시(아래 표) + 한 줄 학습 제안.

## 비범위 (Non-goals)
- 서버·계정·네트워크 전송 없음(전부 로컬, 오프라인).
- 자동 채점 외 새로운 평가 알고리즘 도입 없음.

## 제약 (Constraints) — 반드시 지킬 것
- 순수 HTML5 Canvas, **빌드·런타임 의존성 0**, `file://`에서 동작.
- 기존 입력 모델(`held`/`pressed`)·상태 머신(`game.mode`) 패턴 준수.
- 신규 상태 추가 시 일시정지 메뉴 인덱스(스모크 `PAUSE_ORDER`) 동기화.
- 저사양 태블릿 성능 유지(매 프레임 할당 피하기).

## 기능 요구 (Requirements)
1. 일시정지 메뉴에 **「진단 리포트」** 항목 추가(교사용, 대시보드 근처).
2. 학생(슬롯) 선택 → 진단 화면:
   - 주제별 정답률 막대 + 약점 주제(2시도 이상, 정답률 <60%) 강조.
   - 약점 주제별 **추천 차시·활동** 한 줄(매핑 표 사용).
   - 복습 노트 수, 진행 단계, 퍼즐 선택 회고 요약.
3. **내보내기**: 인쇄/캡처용 텍스트를 클립보드 복사 + (가능 환경) `.txt` 저장.
4. 반 전체 모드: 세 슬롯의 약점 주제 분포 요약 + 공통 약점 Top 3.

## 주제 → 추천 차시 매핑 (초안)
| 주제 키 | 추천 차시(docs/차시별-활동지) |
|---|---|
| privacy, copyright | 1차시 (개인정보·저작권) |
| fake, genai, deepfake | 2차시 (가짜 정보·생성형 AI) |
| bias, filterbubble | 3차시 (공정함·편향) |
| balance, footprint, saving | 4차시 (절제·디지털 발자국) |
| manners, emotion, responsibility, excuse | 5차시 (관계·책임) |

## 수용 기준 (Acceptance — 기계적으로 검증)
- `node tools/validate.js` 통과(데이터·스키마·문서 동기화).
- `npm run test`(스모크+슬롯) 통과 + 신규: 진단 리포트 빌더 순수 함수
  `buildDiagnosticReport(slot)`가 약점 주제·추천 차시를 정확히 매칭하는지 검증.
- 빈 슬롯/통계 없음/전부 정답 등 경계 입력에서 크래시 없음.
- 일시정지 메뉴 인덱스 테스트(`PAUSE_ORDER`) 갱신·통과.

## 모호성 점검 (Ambiguity ≤ 0.2 목표)
- "추천 차시" 매핑 표를 명시했으므로 해소.
- "인쇄"는 브라우저 인쇄/캡처용 텍스트 출력으로 한정(전용 PDF 생성 아님).
- 미해소: 반 전체 PDF 레이아웃 — v1은 텍스트, PDF는 후속.

## 구현 순서 (Execute)
1. `buildDiagnosticReport(slot)` 순수 함수(매핑·약점 추출) + 테스트.
2. `report`/진단 상태 추가, 일시정지 메뉴 연결, 그리기.
3. 내보내기(클립보드/파일) — 기존 `copyTextToClipboard`/다운로드 패턴 재사용.
4. 문서·스크린샷·SW 캐시 갱신.
